Background
We are going to do a simulation of a robot on a grid. The grid will have a pre-designated “goal”
position to be found. The bot will move about on the grid using a fixed set of simple commands
that are provided to the Bot at its construction. There will be a time limit as to how long the
simulation of the bot on the grid is allowed. The simulation ends when either the Bot finds the
goal or when the limit of simulation steps is reached.
Grid
The grid is a square with individual coordinates that we designate as (x,y) that contains a goal
position. We assume that the bottom left corner of the grid is (0,0) and that y is the vertical
direction and x the horizontal direction. Both x and y are greater than or equal to 0. Both the
dimensions of the grid and the goal position are provided as input, and we guarantee that the
provided goal position is within the grid. There is no wrap-around on this grid. Like a Roomba
bot, if a bot moves forward/backward into a wall, its position remains unchanged (it bumps into
the wall and cannot finish the move) and it moves on to executing its next command.
Bot
The bot is very simple, similar to a turtle graphics turtle. A bot has:
• a position in x,y coordinates (the grid)
o the initial position is provided as input, guaranteed to fit on the grid
• a direction the bot faces, represented by four characters: nàNorth, sàSouth, eàEast,
wàWest.
o the initial direction is provided as input.
• four commands it can respond to, also represented as characters:
o l à turn left, r à turn right. The turns do not change the position of the bot, only
its facing direction. A turn results in a change of facing direction to one of the
four cardinal directions: n, s, e, w. Thus if the bot is facing west and turns left, it
will now be facing south without a change of position.
o f à forward, b à backward, move bot one space, forward or backward, in the
direction it is facing. If the bot cannot move because of an obstruction of a wall,
then its position does not change.
o A single string is provided that is a sequence of some combination of these
commands. For example “fblr” is a command sequence. The command string is a
sequence that is repeated. That is, once ‘f’, ‘b’, ‘l’, ‘r’ are each executed, execution
begins again at the start of the sequence, executing ‘f’.
o
Assignment Overview
The main program will read in:
• 3 parameters for a Grid: its dimensions and the x and y positions of the goal
• 4 parameters for the Bot: its initial position as x and y, its initial direction and its
command string.
• a limit for the number of Bot steps to be taken
The main loop will execute as long as the goal has not yet been reached and as long as the
number of Bot steps is less than the limit.
At each Bot step:
• the Bot will execute its next command.
• the count of Bot steps executed will be updated.
At the end (by limit or by reaching the goal, either one), the main will print the final position of
the Bot and whether it had reached its goal.

Things to note
• A Grid instance is a parameter to the Bot constructor. Why is that? The Grid knows its
dimensions and it knows the position of the goal. The Bot can inquire of those values, which
it needs, through its Grid instance.
• Grid needs to come before Bot, since Bot needs a Grid in its construction.
• The Grid is a friend class of the Bot. That means that the Bot has access to the private data
members of the Grid. That will make the code a little easier to write.
• There is a friend function for printing a Bot. It has a very particular format to pass
hackerrank, which is
o (x,y) facing:d Where x,y are its position and d is the direction, a single
character. Get it right!
• There is a commented line in the main loop to print the Bot at the beginning of each cycle.
You should use that (and other output as well) to track how you are doing.
